## PROJECT SUMMARY
A distributed edge-AI traffic management system split across:
  - ESP32-CAM       → camera capture + IR sensors (flash once, sealed)
  - ESP32 Dev Board → LED traffic lights + OLED speed limit display
  - Laptop (Python) → YOLOv8 vision + fusion + decisions + Flask API + Supabase
  - Vercel (Next.js)→ Police dashboard — live feed, overrides, history

=============================================================
SECTION A — HARDWARE CONNECTIONS
=============================================================

--- ESP32-CAM AI-Thinker (SEALED AFTER PHASE 1) ---
Only 3 free GPIOs exist after camera initialisation:

  IR Sensor 1 OUT  → GPIO 2   (25% queue / near lane)
  IR Sensor 2 OUT  → GPIO 4   (50% queue / mid lane)
  IR Sensor 3 OUT  → GPIO 16  (75% queue / far lane)
  All IR VCC       → 3.3V pin
  All IR GND       → GND pin
  Power supply     → 5V / 2A dedicated (NOT from FTDI)

PROGRAMMING (FTDI232 — only during flash):
  FTDI TX → ESP32-CAM GPIO 3 (U0RXD)
  FTDI RX → ESP32-CAM GPIO 1 (U0TXD)
  FTDI GND → GND
  GPIO 0 → GND (bridge during flash only, remove after)

--- ESP32 Dev Board (REPROGRAMMABLE ANY TIME VIA USB) ---
  RED LED    → 220Ω → GPIO 25  (cathode to GND)
  YELLOW LED → 220Ω → GPIO 26  (cathode to GND)
  GREEN LED  → 220Ω → GPIO 27  (cathode to GND)
  OLED SDA   → GPIO 21
  OLED SCL   → GPIO 22
  OLED VCC   → 3.3V
  OLED GND   → GND
  OLED I2C address: 0x3C (try 0x3D if display is blank)

--- IR Sensor Calibration ---
  FC-51 sensor: LOW on OUT pin = vehicle detected, HIGH = clear
  Sensor 1: calibrate for ~1-2 car lengths detection distance
  Sensor 2: calibrate for ~3-4 car lengths
  Sensor 3: calibrate for ~5-6 car lengths

=============================================================
SECTION B — DATA CONTRACTS
=============================================================

--- Command JSON (Laptop → ESP32 Dev Board POST /cmd) ---
  {"signal": "GREEN", "duration": 20, "speed_limit": 40}
  signal:      "RED" | "YELLOW" | "GREEN"
  duration:    integer seconds to hold signal
  speed_limit: 20 | 30 | 40 | 60 | 80  (km/h)

--- Sensor JSON (ESP32-CAM GET /sensors response) ---
  {"s25": 0, "s50": 1, "s75": 0}
  Value 1 = vehicle detected at that distance
  Value 0 = lane clear at that distance

--- Traffic Event Row (written to Supabase) ---
  id, created_at, car_count, queue_level, traffic_score,
  signal, speed_limit, is_override

=============================================================
SECTION C — ALGORITHMS
=============================================================

--- Fusion Formula ---
  queue_level   = (s25 * 1) + (s50 * 2) + (s75 * 3)   [max=6]
  traffic_score = (min(car_count, 20) * 0.6) + (queue_level * 0.4)

--- Decision Table ---
  score < 3     → signal=GREEN,  duration=30, speed_limit=80
  score 3–7     → signal=GREEN,  duration=25, speed_limit=60
  score 7–12    → signal=YELLOW, duration=20, speed_limit=40
  score 12–17   → signal=RED,    duration=20, speed_limit=30
  score >= 17   → signal=RED,    duration=30, speed_limit=20

--- Override Logic ---
  Override lasts 60 seconds from time of POST
  During override: skip fusion/decision, use override values
  After expiry: AI automatically resumes
  Manual clear: POST /clear_override

=============================================================
SECTION D — HTTP API ENDPOINTS
=============================================================

ESP32-CAM (port 80):
  GET  /stream    → JPEG image/jpeg (one frame per request)
  GET  /sensors   → {"s25":int, "s50":int, "s75":int}

ESP32 Dev Board (port 80):
  POST /cmd       → body: command JSON
                 → response: {"status":"ok"} or {"error":"..."}

Laptop Flask API (port 5000):
  POST /override       → {"signal":"RED","speed_limit":20}
  POST /clear_override → {}
  GET  /status         → current system state JSON

SECTION E — SUPABASE SCHEMA
=============================================================

Table: traffic_events
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid()
  created_at   timestamptz DEFAULT now()
  car_count    integer
  queue_level  integer
  traffic_score float8
  signal       text
  speed_limit  integer
  is_override  boolean DEFAULT false
  REALTIME: ENABLED

Table: system_state
  key          text PRIMARY KEY
  value        text
  updated_at   timestamptz
  Rows used:
    key="heartbeat"  → value=ISO timestamp (updated every 5s)
    key="tunnel_url" → value=Cloudflare HTTPS URL for camera stream
